# CWE-94 / CWE-1336 Notes (SSTI no Endpoint de Demo)

## Objetivo
Registrar de forma técnica e rastreável a fraqueza inserida no endpoint `/demo/search`, sua classificação em CWE, e como isso gera tráfego anômalo útil para avaliação do autoencoder.

## Fraqueza Inserida
- **Endpoint:** `GET /demo/search?q=<termo>`
- **Arquivo:** `hub/demo_flask.py`
- **Ponto vulnerável:** uso de input do usuário em template dinâmico:
  - `Environment().from_string(f"Resultado para: {query}")`

Esse padrão permite que payloads com sintaxe de template sejam interpretados no servidor.

## Classificação CWE
- **CWE-1336:** Improper Neutralization of Special Elements Used in a Template Engine (Template Injection).
- **CWE-94:** Improper Control of Generation of Code ('Code Injection').

Relação prática no projeto:
- `CWE-1336` é a classificação mais direta para o caso (SSTI/Jinja).
- `CWE-94` pode ser citado como classificação relacionada quando a injeção de template evolui para execução de lógica não prevista no lado servidor.

## Uso Normal vs Uso Anômalo
Uso normal do endpoint:
- Busca textual rápida para troubleshooting operacional no painel legado.
- Exemplo: `q=lock_1`, `q=motion_detected`.

Uso anômalo/exploratório:
- Queries com delimitadores e expressões de template (`{{ ... }}`, `{% ... %}` etc.).
- Essas queries tendem a ter estrutura e distribuição de caracteres diferente do tráfego legítimo.

## Conexão com o Autoencoder
O objetivo aqui não é “detectar CVE por assinatura”, e sim:
- gerar requisições HTTP com padrão estatístico atípico em relação ao baseline;
- alimentar extração de features de fluxo/tráfego;
- verificar se o autoencoder marca esse comportamento como anomalia.

Em termos de features, esses requests podem influenciar:
- comprimento e variabilidade do query string;
- frequência de caracteres especiais;
- padrões de burst/repetição em tentativas de exploração;
- latência/resposta quando há avaliação de template.

## Exemplos de Requisição para Cenário de Teste Controlado
Normal:

```bash
curl -s "http://localhost:8000/demo/search?q=lock_1"
```

Anômalo:

```bash
curl -s "http://localhost:8000/demo/search?q={{7*7}}"
```

## O Que Pode Ser Afirmado no Relatório
Formulação recomendada:

> Foi introduzida deliberadamente uma fraqueza de template injection no endpoint de demo (`/demo/search`), classificada como `CWE-1336` (com relação a `CWE-94`), para gerar tráfego HTTP anômalo e avaliar a sensibilidade do autoencoder a padrões fora da distribuição de treino.

## O Que Evitar Afirmar Sem Evidência Adicional
- Não afirmar impacto máximo (ex.: RCE completa) sem prova experimental específica.
- Não afirmar que toda detecção do autoencoder equivale a exploração bem-sucedida.
- Não afirmar cobertura total para qualquer payload/ferramenta.
